# CLAUDE.md

## AI Engineering Rules

- Keep the codebase modular and enterprise-friendly.
- Do not mix ERP, CRM, HRM and AI domain logic in one large service.
- Prefer typed DTOs, shared contracts and explicit module boundaries.
- Every user-impacting action should be audit-log ready.
- Multi-tenant context must be preserved across services.
- UI should feel premium, dense, clean and operational rather than marketing-heavy.
- Security changes must consider RBAC, tenant boundaries and audit history.
